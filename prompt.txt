STEP 1 — PREPARE  (build the fleet, hold it idle; NO work runs yet)
--------------------------------------------------------------------------------
  cd e2e/distributed
  LABEL=<ARM>-<BENCHMARK>-full-$(date +%m%d-%H%M%S)      # unique per run
  nohup env \
    FLY_ORG=vamsee-k-933 \
    ARM=<ARM> BENCHMARK=<BENCHMARK> \
    LABEL=$LABEL MACHINES=<WORKERS default 8> \
    SUITE=full \
    MAX_FAILURE_RERUN=<MAX_FAILURE_RERUN> \
    ARCHIVE_TIGRIS=1 TIGRIS_BUCKET=swebench-dist-archive \
    [BENCHMARK=terminal only: TB_TASKS_DIR="$PWD/out/tb21-tasks" — DERIVED CONFIG #4] \
    [per-ARM extras — DERIVED CONFIG #5] \
    [IMAGE=<full registry.fly.io ref> only if reusing a prebuilt image — else omit to bake fresh] \
    ./run-distributed.sh prepare > out/$LABEL.prepare.log 2>&1 & disown

  HARNESS_HOOKS is left UNSET here on purpose — it defaults to 1 (the single
  UNIVERSAL harness) for every claude-* arm x BENCHMARK automatically (HARD RULE 10).
  Only pass it explicitly if you want to OVERRIDE that default (e.g. HARNESS_HOOKS=0
  for a baseline-control run) — and if you do, pass the SAME value on both `prepare`
  and `run`. Leave ESCALATION_PANEL unset. Harmless no-op for econ.

  `prepare` builds/pulls the image, creates the coordinator (COORD_ARMED=0 → /claim
  holds at {wait}) and <WORKERS> warm-idle workers, and SEEDS all tasks. It does NOT
  run, poll, or tear down — it holds.

  The run's app (`swebench-dist-<ARM>-<slug>`, slug = benchmark's `_`→`-` and `verified`→`verif`
  — fly blocks app names containing "verified") is auto-created
  here if it doesn't exist yet, and its Tigris AWS_* secrets are auto-staged from
  `.env.tigris` (run `./provision-tigris.sh` once beforehand if it doesn't exist).
  Each ARM×BENCHMARK combo owns its own app, so this is a fully standalone trigger —
  a different combo can be prepared/run at any other time, fully in parallel.

  Wait for prepare to finish, then confirm readiness from /status:
    - coordinator created; workers_seen == WORKERS; armed == false;
    - counts.pending == <total task count for the benchmark>; 0 dead/failed.
  (Poll the prepare log / coordinator /status; reuse the status.py-style extractor.)

STEP 3 — RUN  (on my go-ahead: arm, process ALL tasks to completion, archive)
--------------------------------------------------------------------------------
  cd e2e/distributed
  nohup env FLY_ORG=vamsee-k-933 \
    ARM=<ARM> BENCHMARK=<BENCHMARK> LABEL=$LABEL SUITE=full \
    ARCHIVE_TIGRIS=1 TIGRIS_BUCKET=swebench-dist-archive \
    [SAME per-BENCHMARK / per-ARM extras as STEP 1 — TB_TASKS_DIR for terminal, etc.] \
    [IMAGE=<same ref> if one was passed at prepare] \
    ./run-distributed.sh run > out/$LABEL.run.log 2>&1 & disown

  `run` RE-RESOLVES the benchmark and task ids on the host — it is NOT a bare attach.
  Passing only LABEL silently defaults BENCHMARK back to verified (wrong app) and,
  for terminal, dies at id resolution (0 ids). Mirror the STEP 1 env (MACHINES may
  be omitted; if HARNESS_HOOKS was overridden at prepare, pass the same value here).

  `run` attaches to the prepared fleet, POSTs /arm (workers begin claiming), raises a
  dedicated GPU only if DEDICATED_CONDUCTOR=1, then polls until the queue drains,
  grades, bundles, archives to Tigris, and tears the fleet down.
    - Monitor to completion (status.py; watch dead / failed / "No space left").
    - EXECUTION failures (no/empty patch, crash, timeout → status=failed) auto-rerun
      up to MAX_FAILURE_RERUN and a post-drain pass — the coordinator handles these;
      a success overwrites the failure in place.

--------------------------------------------------------------------------------
STEP 4 — GRADE-FAILURE RERUN  (GRADE_RERUN=1; explicit pass — NOT a coordinator knob)
--------------------------------------------------------------------------------
  IMPORTANT: the built-in retry covers ONLY execution failures. GRADED failures
  (a real patch that scored resolved=0 → status='done') are TERMINAL in the
  coordinator, so honor GRADE_RERUN with an explicit rerun pass:

  1. From the drained coordinator (or the pulled bundle's queue.db / merged report),
     collect instance_ids with status='done' AND resolved=0 = genuine grade failures.
     For live_verified, DROP the known-ungradeable pylint ids (9771/9772/9785); if any
     other id looks suspicious, gold-grade it (scratchpad goldgrade driver) before
     counting it a real failure.
  2. If there are any and GRADE_RERUN budget remains (default 1): write the failed ids
     to a TASKS_FILE and launch ONE rerun fleet for just those ids with the SAME env:
       LABEL=$LABEL-graderr-$(date +%H%M%S)
       nohup env FLY_ORG=vamsee-k-933 ARM=<ARM> BENCHMARK=<BENCHMARK> \
         LABEL=$LABEL-graderr-... MACHINES=<min(WORKERS, #failed)> \
         TASKS_FILE=<failed.ids> MAX_FAILURE_RERUN=1 \
         ARCHIVE_TIGRIS=1 TIGRIS_BUCKET=swebench-dist-archive \
         [IMAGE=<same ref> if reused] \
         ./run-distributed.sh > out/$LABEL-graderr.log 2>&1 & disown   # all-in-one
     (terminal: TASKS_FILE bypasses suite id-resolution, so TB_TASKS_DIR is not
     required on a rerun fleet — harmless to include.)
     A resolved=1 on rerun supersedes the earlier resolved=0 for that id.
  3. Archive the rerun and VERIFY object count > 0 in Tigris.

--------------------------------------------------------------------------------

--------------------------------------------------------------------------------
  - Combined scoreboard: resolved/total and % (main run, with grade-rerun applied),
    per-instance verdicts for failures, real LiteLLM cost total + by-tier
    (conductor/oracle/reasoner/executor), turns.
  - Tigris: list the archived labels (main + any graded-rerun) and confirm each has
    objects. Retrieve later with:
      ./tools/tigris-archive.sh list | overview <label> | get <label>
    Non-verified benchmarks get a run_id SUFFIX: the label to look up is
    "<LABEL>-<benchmark>" (verified has NO suffix).
  - Confirm no leftover fly machines on the run's app (swebench-dist-<ARM>-<slug>) (cost hygiene).
  - Save a short result memory if anything non-obvious surfaced.
